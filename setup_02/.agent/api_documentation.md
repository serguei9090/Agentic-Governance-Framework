# API Documentation Standards

## Specification
-   **Standard:** OpenAPI 3.0 (Swagger)
-   All backend endpoints MUST have a definition.

## Requirements
-   **Paths:** Clearly defined resource paths.
-   **Methods:** Correct HTTP verbs (GET, POST, PUT, DELETE).
-   **Responses:** Document Success (200, 201) and Error (400, 401, 403, 404, 500) codes.
-   **Schemas:** Request and Response bodies must be typed.
