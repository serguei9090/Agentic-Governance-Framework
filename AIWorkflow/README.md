# Antigravity: AI Workflow Interoperability & Evolution Framework

> **Standardizing the future of agentic collaboration.**

Antigravity is a high-performance, MD-based framework designed to solve the challenges of **AI Workflow Integration, Interoperability, and Evolution**. It provides a structured ecosystem where diverse AI agents can collaborate seamlessly through standardized protocols, shared behavioral laws, and modular capabilities.

---

## 🚀 Core Pillars

### 1. Seamless Integration
Antigravity orchestrates a powerful "Triad" of AI layers, ensuring that the right intelligence is applied to the right task at the right time.
- **Antigravity (Orchestrator)**: The central command center for context management and strategic task decomposition.
- **Gemini CLI (Execution)**: Specialized subagents for high-speed local tasks like linting, formatting, and surgical code edits.
- **Jules CLI (Autonomy)**: An asynchronous remote agent called directly via the CLI (after committing and pushing the codebase) for complex refactors and recursive autonomous loops.

### 2. Universal Interoperability
The framework is built on a "Portability First" philosophy. By standardizing the `.agents/` structure, the entire workflow intelligence can be teleported between projects with zero friction.
- **Standardized Rules**: Behavioral guardrails that translate across different models and environments.
- **Modular Skills**: Encapsulated capabilities (like diagramming or auditing) that can be shared and reused.
- **Portable Workflows**: Executable meta-processes defined in Markdown.

### 3. Recursive Evolution
Antigravity is a **living agentic ecosystem** that learns and adapts. It doesn't just execute tasks; it improves its own operational logic.
- **Self-Correction**: Post-session reflection via the `/auto-improve` workflow to eliminate instruction ambiguity.
- **Strategic Evolution**: A 4-phase growth engine (`/self-evolve`) that refines architecture and expands capabilities as the project matures.

---

## 📂 Project Architecture

The workspace is organized into a modular hierarchy to minimize "Token Friction" and maximize agent efficiency:

- **`.agents/rules/`**: Core behavioral laws and quality guardrails.
- **`.agents/workflows/`**: Executable meta-processes (Auto-Improve, Evolution, Git-Save).
- **`.agents/skills/`**: Modular capabilities (Diagrams, Auditing, Handover).
- **`.gemini/agents/`**: Specialized subagent profiles for local execution.
- **`docs/architecture/`**: High-level architectural guides and the "Triad" north star.
- **`docs/evolution/`**: Historical evolution reports and strategic frameworks.
- **`docs/track/`**: Transparent task tracking and state persistence.

## 🏁 Getting Started

To integrate this framework into your project:

1. **Clone/Copy**: Copy the `.agents/` and `.gemini/` directories to your root.
2. **Initialize**: Use the [triad.md](file:///i:/01-Master_Code/General/AIWorkflow/docs/architecture/triad.md) as your architectural north star.
3. **Execute**: Start your first task and watch the triad evolve.
